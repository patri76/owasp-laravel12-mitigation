La vulnerabilità rientra in OWASP A05:2021 - Security Misconfiguration.
Nel progetto Laravel il debug mode era configurato in modo insicuro. Se APP_DEBUG è impostato su true in ambiente di produzione, l’applicazione può mostrare stack trace, percorsi dei file, variabili di ambiente e dettagli interni utili a un attaccante.

Mitigazione:
- disattivato il debug mode impostando APP_DEBUG=false;
- configurato l’ambiente di produzione con APP_ENV=production;
- aggiornato la configurazione di esempio .env.example;

In questo modo, eventuali errori non mostrano più informazioni sensibili all’utente finale.

es 1.
Ho trovato una vulnerabilità in:

app/Http/Controllers/ArticleController.php

credo sia SQL Injection, perché c’è: whereRaw(...)

Ho clonato progetto, creato database e fatto seed

Nel progetto ho trovato una vulnerabilità di sicurezza all’interno del controller ArticleController, in particolare nel metodo search().
Ho trovato questa query:

$articles = Article::whereRaw("title like '%{$request->search}%'")->get();

Questa implementazione è vulnerabile perché utilizza whereRaw(), che permette di inserire direttamente l’input dell’utente all’interno della query SQL senza alcun controllo.

Credo sia corretto dire che l’uso di query raw espone l’applicazione a SQL Injection:un attaccante può inserire codice SQL malevolo nel campo di ricerca per manipolare la query, accedere a dati sensibili o modificarli.

Ho corretto la vulnerabilità sostituendo l’uso di query raw con query sicure tramite Eloquent. Questo protegge il database da attacchi di tipo injection.
